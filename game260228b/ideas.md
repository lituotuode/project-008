# Bulls and Cows 游戏设计理念

## 设计方向选择

经过深思熟虑，我选择了以下赛博朋克设计方案作为本项目的核心视觉风格。

---

## 选定方案：深度赛博朋克 + 极简主义

### 设计运动（Design Movement）
**赛博朋克极简主义（Cyberpunk Minimalism）**
- 融合 80/90 年代赛博朋克美学与当代极简设计
- 强调高对比度、发光效果和几何线条
- 避免过度装饰，保持功能性和可读性

### 核心原则（Core Principles）

1. **高对比度优先**：深色背景（#0a0a0a）搭配荧光色（霓虹蓝、霓虹紫、科技绿），确保视觉冲击力
2. **功能即美学**：每个 UI 元素都应该有明确的交互目的，装饰性设计不超过 20%
3. **发光与深度**：使用 box-shadow 和 glow 效果营造科幻仪表盘感，避免平面设计
4. **精准排版**：使用等宽字体（如 IBM Plex Mono）强调"代码"感，配合无衬线字体（如 Sora）提高可读性

### 色彩哲学（Color Philosophy）

**主色板：**
- **背景色**：`#0a0a0a`（深黑）- 营造夜间城市感
- **主色调**：`#00d4ff`（荧光蓝）- 代表数据流、网络、科技
- **辅色调**：`#ff006e`（霓虹紫/粉）- 强调重要交互和成功状态
- **强调色**：`#39ff14`（科技绿）- 用于正确答案、成功提示
- **中性色**：`#1a1a1a`（深灰）、`#333333`（浅灰）- UI 容器和分割线

**情感意图：**
- 蓝色 = 冷静、科技、数据感
- 紫/粉 = 能量、刺激、游戏感
- 绿色 = 成功、正确、积极反馈
- 黑色 = 神秘、专业、沉浸感

### 布局范式（Layout Paradigm）

**非中心对称的分层布局：**
- 顶部：游戏标题和统计信息（固定条）
- 中部：核心游戏区域（大型数字输入和结果显示）
- 底部：历史记录和分享功能（可滚动）
- 右侧：新手教程和设置（浮窗或侧边栏）

**移动端适配：**
- 采用纵向堆叠布局
- 保留顶部导航条
- 中部游戏区域占据主要屏幕空间
- 底部操作区域固定

### 签名元素（Signature Elements）

1. **发光边框（Glow Border）**：
   - 所有交互元素（按钮、输入框）使用 `box-shadow: 0 0 20px rgba(0, 212, 255, 0.5)`
   - 悬停时增强发光效果
   - 创造"通电"的感觉

2. **网格背景（Grid Background）**：
   - 使用 SVG 或 CSS 渐变创建细微的网格纹理
   - 增加深度感和科幻感
   - 不抢占主要内容

3. **数字滚动动画**：
   - 数字输入框内的数字切换时，采用垂直滑动动画
   - 猜中时，数字爆炸成粒子效果
   - 增强游戏的交互反馈

### 交互哲学（Interaction Philosophy）

- **即时反馈**：每次点击都有视觉反应（发光、颜色变化、动画）
- **渐进式教学**：首次进入时自动展示教程，不强制但引导用户
- **游戏化反馈**：
  - 正确猜测：绿色发光 + 粒子爆炸
  - 错误输入：红色闪烁 + 震动效果
  - 成功通关：紫色烟火 + 胜利音效（可选）

### 动画指南（Animation Guidelines）

1. **数字输入动画**：
   - 数字切换：`transform: translateY(-100px)` + `opacity: 0` → `translateY(0)` + `opacity: 1`，时长 200ms
   - 缓动函数：`cubic-bezier(0.34, 1.56, 0.64, 1)`（弹性缓动）

2. **按钮交互**：
   - 悬停：`scale(1.05)` + 增强发光
   - 点击：`scale(0.95)` + 闪烁效果
   - 时长：150ms

3. **结果显示**：
   - 成功：绿色发光 + 粒子爆炸（500ms）
   - 失败：红色闪烁（3 次，每次 100ms）

4. **页面过渡**：
   - 淡入淡出：`opacity: 0` → `opacity: 1`，时长 300ms
   - 从下向上滑动：`translateY(20px)` → `translateY(0)`，时长 300ms

### 排版系统（Typography System）

**字体选择：**
- **显示字体（Display）**：`IBM Plex Mono` 或 `Courier Prime`
  - 用于：游戏标题、大号数字、关键数据
  - 权重：700（Bold）
  - 大小：2.5rem ~ 4rem

- **正文字体（Body）**：`Sora` 或 `Inter`
  - 用于：说明文字、按钮、菜单
  - 权重：400 ~ 600
  - 大小：0.875rem ~ 1.125rem

- **代码字体（Code）**：`IBM Plex Mono`
  - 用于：数字输入框、结果显示
  - 权重：500 ~ 700
  - 大小：1.5rem ~ 2rem

**排版层级：**
```
H1（游戏标题）：4rem, Bold, 等宽字体, 荧光蓝
H2（游戏状态）：2rem, Bold, 等宽字体, 白色
H3（标签）：1.25rem, 600, 无衬线, 灰色
Body（说明）：1rem, 400, 无衬线, 浅灰
Small（提示）：0.875rem, 400, 无衬线, 中灰
```

## 设计决策理由

1. **为什么选择深黑背景？**
   - 减少眼睛疲劳（适合长时间游戏）
   - 突出荧光色的视觉冲击力
   - 符合赛博朋克美学

2. **为什么使用等宽字体？**
   - 强化"黑客"或"代码破译"的感觉
   - 数字显示更清晰
   - 增加专业感

3. **为什么强调发光效果？**
   - 营造"通电"的科幻感
   - 提高交互反馈的明确性
   - 符合 80/90 年代赛博朋克审美

4. **为什么采用粒子爆炸动画？**
   - 增强成功时的满足感
   - 提高游戏的趣味性
   - 符合现代游戏设计趋势
